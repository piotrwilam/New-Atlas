# Plan: `6_ablation.ipynb` — Causal Ablation Validation

## Context

The New-Atlas pipeline (notebooks 1A→5) identifies "concept-only" neurons in Qwen2.5-Coder-7B: MLP output dimensions that fire consistently for a syntactic concept (e.g., `assert`) but NOT for the concept's keyword token in non-concept contexts. This classification is purely correlational. The ablation notebook provides **causal validation**: if zeroing these neurons reduces the model's probability of producing the keyword token, they're causally necessary, not just correlated.

## Key Finding from Data Exploration

**The ε threshold direction is reversed from CSP expectations:**

| Threshold | Import concept_only/layer | Assert concept_only/layer | len concept_only/layer |
|---|---|---|---|
| ε=0.5, C=0.8 | 153 | 133 | 7 |
| ε=0.1, C=0.8 | 14 | 38 | 0.5 |

Higher ε gives **more** concept-only neurons on Qwen (not fewer). Reason: higher ε is stricter on the checker mask too, so fewer neurons pass the checker test, leaving more universal neurons as "concept-only." At ε=0.1, most concept/layer combos for builtins and low-fraction ASTs have 0-1 concept-only neurons — too sparse for meaningful ablation.

**Recommendation:** Run the primary experiment at ε=0.5/C=0.8 only. Include ε=0.1 as a secondary comparison but expect many skipped layers.

## Architecture Details

- **Model:** Qwen2.5-Coder-7B, 28 layers, SwiGLU MLP
- **MLP output dimension:** 3584 (confirmed: max neuron index = 3583)
- **Hook target:** `model.layers.{L}.mlp` — captures post-down_proj output `[batch, seq, 3584]`
- **"Neuron" = residual-stream dimension** contributed by that MLP layer, not a SwiGLU gate unit

### Cell 6 — AblationHook class (NEW CODE)
Core intervention mechanism. Key design:
- `register_forward_hook` on `model.layers.{L}.mlp`
- Hook clones output tensor, zeros (or mean-replaces) specified indices along dim=2
- Returns modified tensor (PyTorch replaces the original output)
- Supports `mode="zero"` and `mode="mean"` for SwiGLU robustness check
- Follows same module-lookup pattern as `ActivationExtractor._make_hook`

### Cell 7 — Log-probability extraction (NEW CODE)
Function `get_logprob_at_target(model, tokenizer, prompt, target_token_id)`:
1. Tokenize prompt, find first occurrence of `target_token_id` in the token sequence
2. Run forward pass, get `outputs.logits` (this is new — existing code never captures logits)
3. Compute `log_softmax(logits[0, target_pos - 1, :])` and return `log_p[target_token_id]`
4. Return `None` if keyword token not found in prompt (skip that prompt)

### Cell 8 — Control condition sampling
- `sample_shared_neurons`: sample `n` from the `both` set (or use all if smaller)
- `sample_random_neurons`: sample `n` from complement of `concept_only ∪ both ∪ token_only`

### Cell 9 — Compute mean activations for mean-ablation mode
Use existing `ActivationExtractor` to collect mean MLP output per layer across ~200 random prompts. Required for `mode="mean"` ablation.

### Cell 10 — Main ablation loop
For each concept × layer × condition:
1. **Clean pass** (once per concept/layer, shared across 3 conditions)
2. **Ablated pass** with hook registered → record log P(keyword)
3. **Compute Δ** = clean_logp - ablated_logp per prompt
4. Store results with all metadata

**Both ablation modes:** Run `mode="zero"` and `mode="mean"` for each condition. This doubles the ablated passes but gives the SwiGLU robustness comparison.

**Batched inference:** Batch 50 prompts into groups of 8-16 with left-padding + attention mask. Each prompt's target position is tracked individually. Numerically identical to single-prompt passes, ~4-8x faster.

**Scale:** 10 concepts × 28 layers × (1 clean + 6 ablated [3 conditions × 2 modes]) × 50 prompts ≈ 98,000 forward passes for ε=0.5. ~2-4 hours on A100 with batching.

**Checkpoint:** Save partial results every 2 concepts to resume on interruption.

### Cell 11 — Statistical tests
For each concept/layer: paired t-test and Wilcoxon signed-rank (one-sided: concept_only > random). Bonferroni correction for 28 layers. Output: p-values, effect sizes.

### Cell 12 — Visualization (4 figures)

## Critical Design Decisions

### 1. Per-layer ablation, not all-at-once
Ablate one layer at a time. This gives a causal layer profile directly comparable to the correlational layer profile from notebooks 4-5. All-at-once would be a sledgehammer test.

### 2. SwiGLU and what "zeroing a neuron" means here
The hook captures the MLP module's final output (post-down_proj), shape `[batch, seq, 3584]`. Each "neuron" is a dimension of this output — the MLP's contribution to that residual-stream dimension. Zeroing it removes the MLP's contribution entirely. This is NOT zeroing a SwiGLU gate unit (which would require hooking inside the MLP). This is the standard intervention for transformer MLP ablation studies. Both zero-ablation and mean-ablation are run: mean-ablation replaces with population mean, keeping the output closer to typical distribution (less OOD push from SwiGLU).

### 3. Target token position
Search for the first occurrence of the keyword token ID in the tokenized prompt. Measure `log P(keyword)` at position `target_pos - 1` (autoregressive: position i-1 predicts token i). Skip prompts where keyword is not found as a standalone token.

### 4. Clean pass sharing
One clean forward pass per concept/layer, reused for all 3 condition comparisons. Valid because clean pass has no intervention hooks.

### 5. Shared control may be smaller than concept-only set
When `|both| < |concept_only|`, use all shared neurons. This makes the comparison conservative (smaller "dose" of ablation for shared). Note the size mismatch in results.

## Resolved Questions

1. **Prompt parquet:** `1A_object_prompts.parquet` is available locally in DATA_DIR.
2. **`def` tokenization:** Verified clean. All 11 keywords (`import`, `assert`, `break`, `for`, `while`, `try`, `if`, `def`, `lambda`, `len`, `range`) are single tokens in Qwen's BPE vocab. `def` (id=750) does NOT match inside `default` (different token). Same for `for`/`format`, `if`/`iffy`, `import`/`important`. No false positive risk.
3. **Batching:** Batching (8-16 prompts per GPU call) does NOT change results — computation is identical per-prompt, only GPU utilization improves. Left-padding + attention mask handles variable lengths. Implement batched inference from the start for ~4-8x speedup.
4. **Mean-ablation:** Include in the main run (both `mode="zero"` and `mode="mean"`). This is a pilot — run both ablation modes. Adds ~50% more forward passes but gives the SwiGLU robustness comparison upfront.

## Verification Plan

1. **Sanity check (before full run):** Run ablation on `ast__Assert` at layer 14 only, all 3 conditions. Verify Δ_concept > Δ_random > 0. If not, debug hook placement.
2. **Negative control:** Builtins (`len`, `range`) should show near-zero effect sizes (Δ_concept ≈ Δ_random) because their concept-only sets are tiny (mean 7-8 neurons).
